Claude Code

Build apps, websites, and cool stuff with AI — no coding experience needed.

01

Create a Claude Account

First things first — you need a Claude account.

Pro Plan Required

  1. Go to claude.ai
  2. Click Sign up and create your account (you can use Google or email)
  3. Once signed in, subscribe to the Pro plan ($20/month) — Claude Code requires a paid plan

The Pro plan gives you access to Claude Code plus the chat app. You can cancel anytime.

02

Open Terminal

Press Cmd + Space to open Spotlight. Type Terminal and press Enter.

A window with a blinking cursor appears. This is where you'll paste the first few setup commands. To paste: Cmd + V.

We'll switch to VS Code on the next page — it's much nicer to work in.

Quick Jargon Check

If you're new here, these words might sound scary. They aren't:

  • Terminal: An app where you type commands to your Mac instead of clicking buttons.
  • Folder / Directory: These are the same thing. mkdir means "make folder."
  • Command: A specific line of text you paste into Terminal to make it do something.
  • CLI: Stands for "Command Line Interface"—it's just a program that runs inside Terminal.
03

Install Homebrew

Homebrew is a tool that lets you install other tools. Paste this and press Enter:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your Mac password (the one you use to log in). You won't see characters as you type — that's normal. Type it and press Enter. Then wait.

After it finishes

Look at the last few lines of output. It will say "Next steps" and show you two commands to run. They look like this — copy and run them one at a time:

echo >> ~/.zprofile && echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

If Homebrew shows slightly different commands, use those instead of the ones above.

04

Install Node.js

Node.js lets you run JavaScript projects on your Mac. You'll need it for most web projects.

brew install node

Wait for it to finish. That's it.

05

Set Up Git & GitHub

Git saves snapshots of your code so you never lose work. GitHub stores your code online. You need both.

Install Git and the GitHub CLI

brew install git gh

Create a GitHub account

Go to github.com and sign up. It's free. Come back here when you're done.

Tell Git who you are

Replace "Your Name" and the email with your actual name and the email you used for GitHub:

git config --global user.name "Your Name"
git config --global user.email "you@email.com"

Log in to GitHub from Terminal

gh auth login

It will ask you questions. Use your arrow keys and Enter to select:

  1. GitHub.com
  2. HTTPS
  3. Yes (authenticate Git with your GitHub credentials)
  4. Login with a web browser

It shows a code. Press Enter to open your browser, paste the code, and click Authorize.

06

Install Claude Code

One command. No other software needed.

curl -fsSL https://claude.ai/install.sh | bash

Wait for it to finish. Then close and reopen VS Code's Terminal (press Ctrl + ` twice — once to close, once to reopen). Verify:

claude --version

See a version number? You're good.

This is the official installer from Anthropic. It auto-updates, so you never have to update manually.

07

Set Up Your Two-Desktop Workflow

This is the best way to work: Claude chat on one screen for questions, VS Code with Claude Code on another for building.

Download the Claude app

Go to claude.ai/download and install the Mac app. Open it and sign in with your Claude account.

Put Claude on its own Desktop

With the Claude app open, hover over the green button in the top-left corner of the window. Click "Enter Full Screen". This moves Claude to its own Desktop.

Put VS Code on its own Desktop

Switch to VS Code (Cmd + Space, type "Visual Studio Code"). Do the same thing — hover the green button, click "Enter Full Screen".

Swipe between them

Now swipe left or right with three fingers on your trackpad to jump between Claude chat and VS Code. This is your workflow:

  • Stuck on something? Swipe to Claude chat and ask
  • Got an answer? Swipe back to VS Code and tell Claude Code what to build

VS Code

Setting up your home base for building projects.

01

Download & Install

VS Code is the world's most popular tool for building apps. It's free and works perfectly with Claude.

  1. Go to code.visualstudio.com
  2. Click the big Download button for Mac.
  3. Open the downloaded .zip file.
  4. Drag the Visual Studio Code app into your Applications folder.
02

Open the Terminal Inside

The Shortcut

Open VS Code, then press Ctrl + ` (the key next to 1).

A panel opens at the bottom. This is your new home. You can type all your commands here instead of the old Mac Terminal app.

03

Install the Claude Code Extension

This is the "magic" part. It lets Claude edit your files and show you the changes live.

  1. In VS Code, click the Extensions icon (four squares) on the left sidebar.
  2. Search for "Claude Code".
  3. Click Install.

Look for the official extension from Anthropic.

04

Open a project folder

Before you start building, tell VS Code which folder to work in.

From VS Code

Go to File → Open Folder and pick the folder you want to work in (or create a new one).

The left sidebar now shows all the files inside that folder. This is your file explorer.

From Terminal

You can also open a folder in VS Code from the terminal:

code ~/my-project

If code doesn't work, open VS Code, press Cmd + Shift + P, type "shell command", and click "Install 'code' command in PATH".

05

Using Claude Code in VS Code

Once the extension is installed, here's how to actually use it.

Start Claude Code

Click the Claude Code icon in the top-right of VS Code (it looks like the Claude sparkle). This opens Claude Code in a terminal panel at the bottom.

You can also open it with the keyboard: Cmd + Shift + P, then type "Claude Code: Open".

Talk to it

Type what you want to build or change in the Claude Code panel at the bottom. Claude reads your files automatically because VS Code already has the folder open.

Add a dark mode toggle to this website

Watch the tabs

When Claude edits a file, it opens as a tab at the top of VS Code. You can:

  • Click any tab to see that file's code
  • See the dots — a white dot on a tab means it has unsaved changes
  • Compare changes — VS Code highlights what Claude changed in green (added) and red (removed)

Approve or reject changes

Claude will ask permission before writing files. You'll see:

  • y — Yes, apply this change
  • n — No, skip it
  • a — Yes to all (trust Claude for this session)
06

Why this is better than plain Terminal

  • Tabs at the top: See every file Claude is working on. Click to jump between them.
  • File explorer: The left sidebar shows your entire project folder.
  • Live changes: When Claude edits a file, you see the changes instantly in the editor.
  • Syntax highlighting: Code is colorful and easier to read than plain Terminal.
  • All-in-one: Your code, your files, and Claude are all in one window.

First Project

Everything's installed. Let's build something.

01

Create a folder

This creates a new folder called "my-first-project" in your home directory:

mkdir ~/my-first-project

mkdir = "make directory" (create a folder). ~ = your home folder.

02

Go into it

This moves you inside the folder you just created:

cd ~/my-first-project

cd = "change directory" (go into a folder). Claude Code only sees files in the folder you're in.

03

Launch Claude Code

claude

First time? It'll open your browser to log in. After that, you'll see a > prompt. You're in.

04

Tell it what to build

Create a simple HTML page that says "Hello World" with nice styling

Claude will ask permission before writing files. Type y to approve.

05

See what you built

Simple HTML file

Exit Claude Code (/exit), then open it in your browser:

open index.html

Full app (React, Next.js, etc.)

Exit Claude Code (/exit), then start the dev server:

npm run dev

It'll show a URL like http://localhost:3000 — open that in your browser. Press Ctrl + C to stop the server.

Not sure how to run something? Ask Claude Code: "how do I run this?"

Daily Basics

The commands you'll use every day once you're set up.

The AI Mindset: Chat vs. Code

It's important to know which Claude to talk to:

  • Claude App / Chat: Best for thinking. Ask it "How should I structure this?" or "What's the best way to build a menu?"
  • Claude Code (Terminal): Best for doing. Tell it "Build that menu we just talked about" or "Fix the bug in this file."

Think of the Chat as your Architect and Claude Code as your Builder.

Start Claude Code

Open VS Code's Terminal (Ctrl + `), go to your project folder, then launch Claude:

cd ~/my-project
claude

Replace my-project with your actual folder name.

Resume where you left off

Closed Terminal? Pick up your last conversation:

claude --continue

Browse past conversations

Pick from a list of previous sessions:

claude --resume

Quick one-off question

Don't want a full session? Get an answer and exit immediately:

claude -p "what does this project do?"

Permissions

Claude always asks before changing files:

yYes, allow this
nNo, don't do it
aAlways allow this type (this session)

Slash commands

Type these at the > prompt:

/helpShow all commands
/clearStart fresh
/compactCompress conversation
/commitCreate a git commit
/costShow how much this session has cost
/exitLeave Claude Code

Exiting

Type /exit or press Ctrl + C twice.

Troubleshooting

If things aren't working as expected, try these common fixes:

"Command not found: claude" Try source ~/.zshrc or restart your terminal.
Permission denied Ensure you're in a folder you have access to. Don't use sudo with Claude.
"Rate limit reached" Claude Code shares your Claude Pro limits. Take a short break or check your plan.
Authentication expired Run claude auth logout then claude auth login to re-authenticate.

Memory

Teach Claude your preferences so it remembers them every time.

What is CLAUDE.md?

A plain text file that Claude reads at the start of every conversation. Think of it as a sticky note you leave for Claude with your preferences and project context.

Where to put them

~/.claude/CLAUDE.mdGlobal — applies to every project
CLAUDE.mdProject root — this project only

Create your first CLAUDE.md

Inside Claude Code, just say:

Create a CLAUDE.md that tells Claude I prefer simple explanations and short code

What to put in it

  • Your preferred way of speaking (e.g. "be concise")
  • How you like your files organized
  • Specific rules: "always add comments to explain things"
  • The name of your project or website

"Remember this"

You can also just tell Claude during a conversation:

Remember that I want my website to have a blue and white theme

Claude saves this to auto-memory and recalls it in future sessions.

Skills

Add-ons that give Claude Code extra abilities — like apps on your phone.

What can skills do?

  • Generate PDFs, Word docs, or spreadsheets
  • Follow specific coding patterns
  • Run specialized workflows (testing, deploying)
  • Connect to external tools & services

Find & install skills

Inside Claude Code, type:

/find-skills

Claude searches for matching skills and helps install them.

Using a skill

Once installed, trigger a skill by typing its slash command:

/commit

Where skills live

~/.claude/skills/Your personal skills — available everywhere
.claude/skills/Project-specific skills

Browse the Skills Marketplace

Go to skills.sh — it's an open directory of community-built skills you can install with one command.

Popular skills include:

  • frontend-design — UI/UX design principles
  • vercel-react-best-practices — React patterns from Vercel
  • web-design-guidelines — Design standards and accessibility

Install any skill directly:

npx skillsadd owner/skill-name

Replace owner/skill-name with the actual skill from skills.sh — each listing shows the exact install command.

Tips & Tricks

Level up your workflow once you're comfortable with the basics.

Plan before building

Press Shift + Tab to enter plan mode. Claude will research your code and propose a plan before writing anything. You approve first, then it executes.

Be specific

"make a login page"
"Create a login page with email and password fields, a submit button, validation errors, using our Button component from src/components"

Make Claude think harder

Add these words to your prompt to control how deeply Claude thinks:

"think" Moderate thinking — good for multi-file changes
"think hard" Deep thinking — for complex bugs or architecture decisions
"ultrathink" Maximum brainpower — for critical, high-stakes decisions

Example: "think hard about why this login page keeps crashing and fix it"

What Claude can read

Claude Code isn't just for writing code. You can throw all kinds of files at it:

  • Spreadsheets & Excel: Analyze data, find patterns, create summaries
  • PDFs: Extract text, summarize documents, compare versions
  • Images & screenshots: Read error messages, interpret charts, explain what's on screen
  • Writing: Draft emails, reports, meeting notes, documentation

Pipe things in

Send files or command output directly:

cat error.log | claude -p "explain what went wrong"

The | ("pipe") sends one command's output into the next.

Prompt Recipes

Try these specific prompts for better results:

Clean up: "Rewrite this file to be easier to read and simpler, but keep it working the same way."
Check: "Check my work for any mistakes and tell me if there's a better way to do it."
Explain: "Explain how this code works like I'm five years old."
Fix: "I'm seeing an error message that says [paste error here]. Can you find out why and fix it?"

Updating

If you used the curl installer (recommended), Claude Code updates itself automatically. If you installed via npm, use:

npm update -g @anthropic-ai/claude-code
